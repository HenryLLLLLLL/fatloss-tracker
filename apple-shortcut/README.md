## 🔧 前置：数据库设置（仅需运行一次）

> 快捷指令现已支持 UPSERT（同一天重复运行不会建重复记录）。需要先在 Supabase 加唯一约束 + 新增字段。

### 1. 加唯一约束

打开 Supabase SQL Editor：
👉 https://supabase.com/dashboard/project/qzmdcdwxleswncfspxkr/sql/new

粘贴以下 SQL，**点 Run**：

```sql
-- training_log：同一天同一训练类型只允许一条记录
ALTER TABLE public.training_log
  ADD CONSTRAINT training_log_date_type_unique
  UNIQUE (date, training_type);
```

> ✅ 运行后显示 `ALTER TABLE` 即成功。如果提示已存在则忽略。

### 2. 新增字段（weight_log 加步数和静息心率）

```sql
-- 给 weight_log 加步数字段（来自 Apple Watch）
ALTER TABLE public.weight_log ADD COLUMN IF NOT EXISTS steps INTEGER DEFAULT NULL;

-- 给 weight_log 加静息心率字段
ALTER TABLE public.weight_log ADD COLUMN IF NOT EXISTS resting_hr_bpm NUMERIC DEFAULT NULL;

-- 给 weight_log 加心率变异性字段
ALTER TABLE public.weight_log ADD COLUMN IF NOT EXISTS hrv_ms NUMERIC DEFAULT NULL;

-- 给 weight_log 加静息代谢字段
ALTER TABLE public.weight_log ADD COLUMN IF NOT EXISTS resting_energy_kcal NUMERIC DEFAULT NULL;
```

> ✅ 如果某列显示「already exists」则忽略，继续下一条即可。

### 3. 修改字段类型（training_log 的数值改为 decimal）

```sql
-- Apple Health 返回的是小数（如 77.877），需要 numeric 类型
ALTER TABLE public.training_log
  ALTER COLUMN duration_min TYPE NUMERIC USING duration_min::NUMERIC,
  ALTER COLUMN estimated_calories TYPE NUMERIC USING estimated_calories::NUMERIC;
```

### 4. 更新 CHECK 约束（加入 apple_watch 类型）

```sql
-- 先删旧的 CHECK 约束
ALTER TABLE public.training_log
  DROP CONSTRAINT IF EXISTS training_log_training_type_check;
-- 再加新的（包含 apple_watch）
ALTER TABLE public.training_log
  ADD CONSTRAINT training_log_training_type_check
  CHECK (training_type IN ('boxing', 'strength', 'boxing_strength', 'apple_watch'));
```

> ⚠️ 以上 4 段 SQL **全部运行完毕**后再开始创建快捷指令！

## 📤 第三部分：发送体重数据（UPSERT）

> ⚠️ 步数总和在第 23 步已经产生，下面第 9 字段可以直接引用！

### 第 24 步：发送体重数据到 Supabase

> ✅ **UPSERT**：同一天重复运行不会建重复记录，而是更新已有记录。

1. 搜索框输入 **「获取 URL 的内容」** → 点它
2. 点 **「方法」** → 选 **「POST」**
3. 点 **「URL」** 输入框，粘贴（注意 URL 末尾加了 `?on_conflict=date`）：
   ```
   https://qzmdcdwxleswncfspxkr.supabase.co/rest/v1/weight_log?on_conflict=date
   ```
4. 点「显示更多」展开

5. 点 **「头部」**，添加以下 3 组键值对：

   | 键 | 值 |
   |---|---|
   | `apikey` | （粘贴 API 密钥） |
   | `Authorization` | `Bearer ` + API 密钥 |
   | `Prefer` | `return=minimal, resolution=merge-duplicates` |

   > ⚠️ `Prefer` 的值有两个，逗号分隔：
   > `return=minimal` = 不返回内容（节省流量）
   > `resolution=merge-duplicates` = 遇到重复 date 时更新而不是报错

6. 点 **「请求体」** → 选 **「JSON」**

7. 在 JSON 区域**逐字段添加**（点「添加新字段」），共 **9 个字段**：

   | # | 键（键盘输入） | 值（选变量） |
   |---|------------------------|----------------------|
   | 1 | `date` | 「选择变量」→ **「格式化日期」**（第 17 步）|
   | 2 | `weight_kg` | 「选择变量」→ 第 3 步的体重值 |
   | 3 | `bodyfat_pct` | 「选择变量」→ 第 5 步的体脂率值 |
   | 4 | `bmi` | 「选择变量」→ 第 7 步的 BMI 值 |
   | 5 | `lean_mass_kg` | 「选择变量」→ 第 9 步的去脂体重值 |
   | 6 | `resting_energy_kcal` | 「选择变量」→ **「静息代谢总和」**（第 15 步的变量）|
   | 7 | `resting_hr_bpm` | 「选择变量」→ 第 11 步的静息心率值 |
   | 8 | `hrv_ms` | 「选择变量」→ 第 13 步的 HRV 值（如跳过则不加此项）|
   | 9 | `steps` | 「选择变量」→ **「步数总和」**（第 23 步）|

   > 💡 字段 3、8 是**可选**的：如果对应的「查找健康样本」步骤被跳过了（健康 App 里没有此类型），就不要添加这个字段。

> ✅ 预期显示：「获取 URL: ...weight_log?on_conflict=date | 方法: POST | 头部: 3 项 | 请求体: JSON → 8-9 个字段」

## 常见问题

### Q: 为什么只采 10 项？之前的 16 项呢？
**A:** 我根据你在 Health App 里的实际数据和减脂目标做了筛选。未采集的指标（站立时间、洗手、步态参数等）对减脂分析几乎没有参考价值，去掉可以大幅减少配置步骤（45 步 → 30 步），降低出错概率。

### Q: 为什么不用「聚合操作」直接求和？
**A:** iOS 26 的「查找健康样本」已移除该选项。改用「重复每个」+「计算」循环累加，效果完全相同。

### Q: 去脂体重（Lean Body Mass）为什么重要？
**A:** 减脂的核心是「减脂肪、保肌肉」。如果你只看体重下降，可能减掉的是水和肌肉。去脂体重稳定或上升 + 体脂率下降 = 正确的减脂方向。

### Q: HRV 和静息心率能说明什么？
**A:** 
- **静息心率持续上升** → 可能过度训练、休息不足、或者压力过大
- **HRV 下降** → 身体恢复能力变差，建议减少训练强度或增加休息
- 这两个指标可以帮助你避免"越减越虚"

### Q: 变量老是选错？
**A:** 建议给关键动作改名。例如第 3 步改名为「体重值」，第 18 步的「查找健康样本」改名为「活动能量样本」。长按动作 →「重新命名」。

### Q: 活动能量的值不对？
**A:** Apple Watch 记录的是「活动能量」（Active Energy），单位是千卡。你截图中是 91.5 kcal（可能还没运动），打完拳后通常在 400-600 kcal。

### Q: 休息日会自动记录训练吗？
**A:** 会，但 `estimated_calories` 和 `duration_min` 会接近 0。看板会识别为休息日。

### Q: 手动记录会和自动数据冲突吗？
**A:** 不会。自动数据用 `training_type=`apple_watch`，手动记录用不同 `training_type`。UPSERT 按 `(date, training_type)` 判断唯一性，互不影响。

### Q: 想改自动运行时间？
**A:** 快捷指令 App →「自动化」→ 点对应自动化 → 改时间 → 完成。

### Q: PWA 看板为什么要输密码？
**A:** PWA 部署在 GitHub Pages 上，任何人知道 URL 都能访问。密码保护是为了防止你的体重、饮食、训练数据被公开查看。

### Q: 健康 App 里找不到某些指标的类型？
**A:** 你的体脂秤 App 可能没有把全部数据同步到 Apple Health。去体脂秤 App 的设置里找「同步到健康」，把能开的开关全部打开。至少保证体重、体脂率、BMI、去脂体重这 4 项有。
